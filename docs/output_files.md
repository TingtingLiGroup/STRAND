# Output Files

This document summarizes common output files generated by STRAND Tools.

---

## 1. Pattern Classification outputs

Main command:

```bash
subcellfeat
```

Main output:

```text
pattern.parquet
```

or:

```text
pattern.csv
```

depending on the extension specified by `--out`.

### Main columns

```text
cell
gene
Bento feature columns
SPRAWL feature columns
p_<class> probability columns
pattern
```

### Feature columns

```text
bento_cell_inner_proximity
bento_nucleus_inner_proximity
bento_nucleus_outer_proximity
bento_cell_inner_asymmetry
bento_nucleus_inner_asymmetry
bento_nucleus_outer_asymmetry
bento_point_dispersion_norm
bento_nucleus_dispersion_norm
bento_l_max
bento_l_max_gradient
bento_l_min_gradient
bento_l_monotony
bento_l_half_radius
sprawl_peripheral
sprawl_central
sprawl_punctate
sprawl_radial
```

### Pattern column

```text
pattern
```

This is the official final localization label.

---

## 2. Compartment Detection outputs

Main command:

```bash
subcellfeat-compartment
```

Common outputs:

```text
*_sampled_batches.csv
*_sampled_batches_meta.json
embedding plots
subdomain plots
cluster assignment tables
```

### `sampled_batches.csv`

Summary table for processed groups or batches.

Typical fields:

```text
dataset
batch/group
status
backend
frac
radius
n_clusters
best_k
n_transcripts
running time
output paths
```

### `sampled_batches_meta.json`

Metadata file recording parameters and processed groups.

---

## 3. Colocalization outputs

Main command:

```bash
subcellfeat-coloc
```

Common outputs:

```text
instant_significant_pairs.csv
prefilter_summary.csv
viz_cpb_heatmap.png
viz_coloc_network.png
top-pair spatial plots
region annotation files
```

### `instant_significant_pairs.csv`

Main table of significant colocalized gene pairs.

Typical columns include:

```text
gene_1
gene_2
cpb_pvalue
score/statistics fields
```

### `prefilter_summary.csv`

Summary of input size and filtering effects.

### Visualization outputs

```text
viz_cpb_heatmap.png
viz_coloc_network.png
viz_top_pair_*.png
```

These files are designed for quick inspection and reporting.

---

## 4. Large optional outputs

The following options may generate large files:

```bash
--save-matrices
--save-pp-pvals
--save-intermediate
--save-qc-details
--export-csv
```

Use them only when detailed debugging or downstream analysis is needed.

---

## 5. Recommended output organization

```text
results/
├── pattern/
├── compartment/
└── colocalization/
```

For large production runs:

```text
Pattern/
Compartment/
Colocalization/
logs/
```

Do not commit large result folders to Git.
